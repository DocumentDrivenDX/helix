# Data Contract authoring prompt

Draft a Data Contract.
Sections in order: Producer, Consumer, Schema Versioning Policy, Semantic Field Definitions, Freshness SLA, Evolution Policy, Termination Clauses.
Make obligations specific, versioned, testable, and owned by named teams.
Put linkages to briefs, inventories, source profiles, or governance decisions in frontmatter `ddx.links`, not the body.
