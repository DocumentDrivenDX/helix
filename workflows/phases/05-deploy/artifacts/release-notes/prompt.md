# Release Notes Generation Prompt

Create concise, user-facing release notes for this release.

## Storage Location

`docs/helix/05-deploy/release-notes.md`

## Key Requirements

- Keep the writing user-focused and scannable.
- Include only the changes, fixes, breaking changes, and upgrade steps users need.
- Prefer benefits and actions over implementation detail.
- Omit generic release-summary filler.

## Completion Criteria

- [ ] Release impact is clear
- [ ] Required user actions are explicit
- [ ] Rollback or known-issue guidance is included when needed
